Welcome to TimeLord

 

 

Getting Started

 

This is a quick guide to get you going.  It is quite brief because, let’s face it, TimeLord is not rocket surgery.

 

Before you start, (at this stage) you must have a JIRA server that you can log into and that has one or more projects defined.  If you just want to see what TimeLord is like, you could use the demo account that is available at:

 

http://www.atlassian.com/software/jira/

 

Or set up an account on their server for only $10 (all monies to charity!).

Running TimeLord

 

1.     Download TimeLord for your operating system (currently only Mac and Windows) from:

 

http://sourceforge.net/projects/tlord/files/

 

2.     Unzip and copy to a convenient location.

3.     Double-click on the TimeLord icon and you should see something like this:

 

 

4.     Just click ‘No’ and click the ‘Configuration’ button as marked above in red.  There is just some ‘valid’ dummy info in there to start things off.

 

Note: if you don’t have a JIRA server, you can continue to use TimeLord as a task timer.  I hope to improve this ability when the current functionality is working well.

Configuring JIRA Access

 

1.     You should now see the Configuration window:


 

 

2.     Enter your JIRA server details IN FULL i.e. https://etc.etc.etc

3.     Username/Password

4.     The project you would currently like to track.  If you have multiple projects, you can swap between them from here but only one project can be monitored at a time.

5.     If you’re happy with your details, just click ‘OK’ and restart TimeLord (I hope to fix it so you don’t have to-it’s on the bug list).  If you want to check you details first, click ‘Try’’ and if the details are ok, you sill see “Success”.  Again, you have to restart TimeLord.

 

 

 

Using TimeLord

 


 

1.     1 and 2:  If the task you want to record is either not in JIRA or is not work related (e.g. lunch) then click the appropriate button.

2.     3:  Successfully logging into your JIRA server should have automatically propagated this drop-down menu with all the tasks assigned to your username.  If you don’t know which task is which, mouse-over a task and a pop-up will appear with the description:

 

 

 

 

 

 

3.     4:  You must enter a description of the work you are about to perform before you can start recording, otherwise, what’s the point?

4.     Once you have entered a description, the start button will be enabled.  Click it (5) and the recording process will begin.

5.     When you have finished your work on that task, click the stop button (5) and the time will be recorded and saved in the database.

Generating a Report

 

1.     Once you have recorded some work, you can generate a report that shows what you have worked on, how long you spent working and your descriptions of the activities involved.  Click the ‘Report’ button marked below in red:


 

 

 

2.     You should now see the ‘Report’ window:


 

 

3.     1:  Choose the time interval you would like a report for.  You can choose from one day, one week, a month or the whole year.

4.     2:  Pick the start date of the interval.  If the interval is big enough, it will stop at your current date.

5.     3:  Choose the JIRA issue you are interested in.  If you do not change this, ALL your work for the given period will be displayed.

6.     4:  A summary of what will be displayed in the report will be shown here.  If it is not what you want, make changes and they will show up.

7.     Click the ‘OK’ button and you should see your report:

 

 

8.     From here you can create a PDF or if REALLY necessary, print out a copy (Save the Planet!).

 

 

 

 

 

Managing the Database

 

Note: This software is offered ‘As Is’.  It is not fully tested and is not offered with any guarantee as to the safety or security of any data entered into it.  At least for now, please duplicate any data entered into TimeLord to avoid possible bitter tears.

 

1.     Click the ‘Configuration’ button to open the Configuration window.

2.     The second half of this window is for managing the TimeLord database that is created when TimeLord is first run:

(~/.TimeLord.db or /Users/<your Windows username>/.TimeLord.db)

 


 

 

3.     1:  WARNING! This will delete your entire database and hence, every entry you have ever made into TimeLord.  I would like to hook a USB Breathalyzer to this button but alas, cannot.  BE WARNED!

4.     2:  Once you have made a backup of your TimeLord database, you can click this button to find the backup and restore your database to the last backed up state.

5.     3:  This is the MOST important button.  Use it often.  It will allow you to create a backup of your database and save it to a USB stick, network server or other form of backup.
